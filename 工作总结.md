# Drew Search — Sprint 1-4 工作总结

## 项目概述

**Drew Search** 是一个分布式 Agent 轨迹搜索引擎，旨在让 AI Agent 通过搜索和复用已有的「任务快照」（.amd 文件），避免重复造轮子。核心价值闭环为：**快照发布 → 向量检索 → 链路复用**。

**仓库地址**: `https://codeup.aliyun.com/5f0af04d5fd102f22f6b747c/MyDrew.git`
**当前阶段**: Phase 1 — MVP 核心闭环（第 1-8 周）
**已完成**: Sprint 1-4（第 1-4 周）

| 组件 | 技术 | 理由 |

### AgentRank 冷启动策略

AgentRank 采用 6 维加权评分，冷启动期（索引量 < 100,000）自动调整权重：

| 维度 | 标准权重 | 冷启动系数 | 说明 |
|------|---------|-----------|------|
| 向量相似度 | 0.35 | 1.0x | 核心语义匹配 |
| 结构完整度 | 0.15 | **2.5x** | 步骤数越多越好 |
| 领域标签匹配 | 0.15 | **3.0x** | Jaccard 相似度 |
| 复用次数 | 0.10 | **0.3x** | 初期数据少，降权 |
| 成功率 | 0.15 | 1.0x | 已验证的可靠性 |
| 官方认证 | 0.10 | 1.2x | 官方快照额外加分 |

## 关键问题与解决

### 1. Go RE2 正则引擎不支持 Lookahead

**问题**: 原始 `cotRegex` 使用了 `(?=...)` 前瞻语法，Go 的 RE2 引擎不支持。
**解决**: 将 lookahead 改为显式的终止符匹配（`\n`` ``` ``、`\n### Step`、`\n## [Step`、`\n# `、`$`），同时改用双引号字符串避免 backtick 冲突。

### 2. CRLF 行尾导致 `\s*$` 跨行匹配

**问题**: Windows 环境生成的 `.amd` 文件使用 `\r\n` 行尾，`\s*` 会跨越 `\n` 导致 tool call 正则无法匹配。
**解决**: 为 tool call 相关正则添加 `(?m)` 多行模式标志，将尾部 `\s*` 改为 `[^\S\n]*`（匹配水平空白但不含换行）。

### 3. 双格式 Step Header 兼容

**问题**: 原始解析器仅支持 `## [Step N: Title]` 格式，但种子数据和生成数据使用 `### Step N: Title`。
**解决**: 扩展 `stepHeaderRegex` 为 `(?:\[Step\s+(\d+)(?::\s*(.+?))?\]|Step\s+(\d+):\s*(.+?))` 同时匹配两种格式，并在 `extractSteps` 中按 group index 分别处理。

### 4. 双格式 Tool Call 兼容

**问题**: 原始格式使用 `**Name:** \`xxx\`` 标记，种子数据使用 `` ```tool_call `` 代码块。
**解决**: 在 `extractToolCall` 中优先匹配 `` ```tool_call `` 块（YAML 解析），回退到 `**Name:**` 行匹配。
