### Sprint 7-8 (第 7-8 周): 集成测试与 MVP 发布

| 任务 | 产出 | 负责角色 | 验收标准 |

| **Blueprint Parser 编译缓存层** | 缓存模块 | 后端 | Task_CID 首次翻译后固化，后续命中延迟 <5ms，缓存命中率 >90% |
| **Fallback 规则解析器** | 降级模块 | 后端 | LLM 服务不可用时仍可解析标准结构 |

> **编译缓存层关键设计**: Task_CID 作为缓存 Key 写入 PostgreSQL JSONB + Redis。新分支创建时仅对差异部分触发增量重编译。缓存容量预估：100 万快照 × 平均 5KB StateGraph ≈ 5GB，完全可内存化。这是控制 LLM Token 成本的核心架构。

## 风险管理

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|--------|
| Embedding 模型对 Agent 任务描述语义理解不足 | 中 | 搜索准确率低 | 收集领域数据微调 BGE-M3；A/B 测试多模型 |
| Eino 框架功能不足或 API 变更 | 中 | Harness 开发受阻 | 设计抽象层解耦；保持 LangGraph 备选 |
| .amd 格式推广困难，Agent 不愿发布 | 高 | 索引量不足 | 提供 SDK 自动发布；激励机制前置 |
| 分布式网络冷启动节点少 | 高 | 网络效应弱 | 先中心化运营，积累足够节点后再去中心化 |
| 微支付 Gas 成本过高 | 中 | 经济模型不可持续 | Launch 前用积分+Stripe；P Last 再引入 Web3，届时选低 Gas 链 |
| Blueprint Parser 幻觉导致错误 StateGraph | 中 | 执行链路异常 | 强制 Schema 校验；沙盒预执行验证 |
| Blueprint Parser 高并发下 Token 成本爆炸 | 中 | 费用失控 | 编译缓存层确保每个 Task_CID 仅翻译一次；缓存命中率目标 >95% |
| AgentRank 冷启动新快照沉底 | 高 | 优质内容不可见 | 动态权重调整；官方认证快照提权；StructuralCompleteness 权重前置 |
| 企业客户对 Web3 支付合规性顾虑 | 高 | B 端落地困难 | Launch 前仅提供 Stripe 法币通道；Web3 延至 P Last，待合规评估完成后启动 |

## 成本估算（MVP 阶段）

| 资源 | 月成本估算 |
|------|-----------|
| 云服务器（开发+测试） | $2,000 - $3,000 |
| GPU 服务器（Embedding 推理） | $1,500 - $2,500 |
| 向量数据库（Qdrant Cloud 或自建） | $500 - $1,000 |
| LLM API（Blueprint Parser） | $1,000 - $2,000 |
| CI/CD + 工具链 | $500 |
| 企业计费网关（Stripe 等） | $200 - $500 |
| **总计** | **$5,700 - $9,500/月** |

> **编译缓存层成本优化说明**: Blueprint Parser 编译缓存上线后，预计 95%+ 的快照检索请求可直接命中缓存（<5ms, 0 Token），仅新入库快照触发 LLM 翻译。LLM API 月成本可从 $2,000 降至约 $300-500。
